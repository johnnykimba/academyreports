The cascade

Marks move in one direction: forward, and only when ready.

01 — TEACHER

Enters marks by subject

Scoped to their own subject and class only. Never sees a student's overall total or another teacher's entries.

→
02 — ADMIN

Verifies, then releases

Nothing reaches a student until admin flips the release switch — per faculty, option, class, subject and term.

→
03 — STUDENT

Views with code + surname

No password to lose. A parent can check results the same way, the moment they're released.

Built on a ledger, not a spreadsheet.
